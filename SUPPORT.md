# Support

## Support Boundary

`background-tasks` is experimental and solo-maintained.

Support is limited to:

- reproducible bugs in supported versions
- documentation corrections
- scoped proposals that fit the project goals
- security reports through the private process in `SECURITY.md`

Support does not include:

- private consulting
- urgent on-call response
- custom feature implementation
- debugging unsupported forks or modified releases
- help with secrets, credentials, or infrastructure not controlled by this project

## Where To Ask

- Bugs: GitHub issues
- Documentation corrections: GitHub issues
- Proposals: GitHub issues
- Security: use `SECURITY.md`; do not open a public issue

## Before Opening An Issue

Include:

- version or commit
- install channel
- operating system and runtime versions
- minimal reproduction
- expected behavior
- actual behavior
- relevant logs with secrets redacted

## Response Expectations

The maintainer reviews issues on a best-effort basis. No response-time SLA is promised unless a separate paid or organizational support agreement exists.
